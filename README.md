Контекст и проблема. Разбить 1 большую задачу - на чёткий план создания решения, прототипа для биомедицины. Структура БЭ (биоэквивалентности) исследований очень стандартизована - есть ГОСТ (какие?), регуляторные докумменты. Из открытых исчточников можно получить параметры РК-параметры, CVIntra, типовые дизайны. Задача - автомматизация БЭ исследований. Этапы исследования БЭ как части Дорожной карты проекта - 1. выбрать корректный дизайн исследования (2*2 cross-over, replicate design, 3-way ) 2. определить уровень внутрисубъектной вариабельности (CVIntro) 3. принять решение о применимости RSABE (Reference-scaled  Average Bioequivalence) 4. рассчитать объём выборки с учётом - статистической мощности, возможных drop-out, screen-fail 5. учесть режим приёма - натощак или после еды 5. обеспечить выполнение и соблюдение международным требованиям (РФ Решение №85) 6. сформировать структурированный синопсис исследования 7. провести первичный поиск литературы по фармокинетике референтного препарата. Предложи структуру - чат-бот, или веб-сервис с полем ввода данных. Связка сервер - RAG База знаний - модуль парсинга данных
