# 🚀 BE Platform — Roadmap

## AI-ПЛАТФОРМА АВТОМАТИЗАЦИИ ИССЛЕДОВАНИЙ БИОЭКВИВАЛЕНТНОСТИ (ОНКОЛОГИЯ)

> Детерминированное расчётное ядро + RAG на GigaChat + Регуляторная экспертиза

---

## ✅ v1.0 — MVP (текущая версия)

### Расчётное ядро (Kotlin + Quarkus)
- [x] **Design Engine** — выбор дизайна: 2×2 crossover / replicate / parallel / 3-way
- [x] **RSABE Module** — определение применимости RSABE (CV > 30%), масштабирование границ
- [x] **PK Module** — расчёт washout, оценка риска накопления, предупреждения по t½
- [x] **Sample Size Engine** — Owen's exact method, итеративный t-тест, коррекция dropout/screen-fail
- [x] **Document Generator** — синопсис в форматах Markdown, JSON

### API (Quarkus REST + OpenAPI)
- [x] `POST /api/v1/calculate` — полный расчёт в одном запросе
- [x] `POST /api/v1/design` — быстрый выбор дизайна
- [x] `POST /api/v1/rsabe` — оценка RSABE
- [x] `POST /api/v1/sample-size` — расчёт объёма выборки
- [x] `POST /api/v1/rag/query` — запрос в RAG-базу знаний
- [x] Swagger UI / OpenAPI 3.0 документация
- [x] JWT Auth + валидация входных данных

### AI-слой (GigaChat RAG)
- [x] Интеграция с GigaChat API (OAuth2)
- [x] Регуляторная база знаний (ЕЭК №85 · EMA · FDA)
- [x] AI-объяснения решений (без влияния на математику)
- [x] Fallback-режим при отсутствии API-ключей

### Frontend (React + TypeScript)
- [x] Дизайн-калькулятор с визуализацией RSABE
- [x] Панель результатов: метрики, PK параметры, расчёт выборки
- [x] Экспорт синопсиса (Markdown, JSON)
- [x] AI-чат с регуляторной базой знаний
- [x] Tailwind CSS UI

### Datasets
- [x] ФК-данные 12 онкопрепаратов (PubMed)
- [x] База Q&A для RAG (12 пар по регуляторике)
- [x] Примеры синопсисов (4 кейса)

## 🔮 v2.0 — AI-агенты для онкологов (Q4 2024 — Q1 2025)

### Интеллектуальные агенты
- [ ] **BE Planner Agent** — автоматическое планирование полного исследования по INN
- [ ] **Literature Review Agent** — первичный поиск литературы по ФК референтного препарата
- [ ] **Regulatory Compliance Agent** — проверка протокола на соответствие ЕЭК №85/EMA/FDA
- [ ] **Statistical Analysis Agent** — пострасчётный анализ: ANOVA, RSABE тест, ДИ
- [ ] **Risk Assessment Agent** — оценка рисков регуляторных замечаний

### Мультиагентная архитектура
```
User Query
    │
    ▼
Orchestrator Agent (LangGraph / CrewAI)
    ├─ PK Literature Agent → PubMed/ClinicalTrials
    ├─ Design Planning Agent → Расчётное ядро
    ├─ Regulatory Check Agent → RAG Base
    ├─ Document Generation Agent → Synopsis Builder
    └─ Risk Analysis Agent → Compliance Check
    │
    ▼
Structured Synopsis + Risk Report
```

## 💡 Концептуальное преимущество

**Мы не строим "ещё один LLM-калькулятор".**

Архитектурное решение:
1. **Детерминированное ядро** — математика БЭ не зависит от LLM (нет галлюцинаций в расчётах)
2. **AI-слой объяснений** — GigaChat объясняет *почему* был выбран дизайн (регуляторный контекст)
3. **Онко-специфичность** — алгоритмы учитывают особенности онкопрепаратов (высокая CV, длительный t½, food effect)
4. **Регуляторная трассируемость** — каждое решение обосновано ссылкой на нормативный документ

> *"BE Platform = статистический движок + регуляторная экспертиза + AI-объяснения"*
