# Тестирование BE Platform — Практические Врачебные Сценарии

## Введение

В данном документе описаны практические врачебные тест-сценарии для платформы автоматизации исследований биоэквивалентности (БЭ) в онкологии. Каждый сценарий отражает реальную клиническую ситуацию, которая может возникнуть при проектировании BE-исследования для онкологических препаратов с высокой вариабельностью.

**Ключевой сценарий (описан в задаче):**

> *Влияние процесса приёмов пищи на биоэквивалентность высоковариабельного онкологического препарата*

---

## Сценарий 1: Сорафениб — влияние пищи при высокой вариабельности

### Исходные данные

| Параметр | Значение |
|---|---|
| Препарат | Сорафениб (Nexavar®, генерик) |
| CV_intra | 42% |
| t½ | 48 ч |
| Подтверждённый эффект пищи | Да (AUC↑ 29% при высококалорийной пище) |
| Популяция | Онкологические пациенты (гепатоцеллюлярная карцинома) |

### Методика

1. **Дизайн**: Реплицированный (4-период, 2-последовательность) — TRTR/RTRT
2. **Состояния**: Натощак + После высококалорийной пищи (>800 ккал, >50% жиров)
3. **Washout**: 10 дней (5 × t½ = 5 × 48 = 240 ч)
4. **Выборка**: N = 64 рандомизировано, N = 92 скринировано

### Цель

Доказать биоэквивалентность генерика сорафениба оригинальному препарату как натощак, так и в состоянии после приёма жирной пищи, используя RSABE-критерии для высоковариабельного препарата.

### Как работает модель

- CV_intra = 42% > 30% → HVD → RSABE применимо
- Масштабированные границы: ~69.8–143.2% (динамически по σ_R)
- Критерий: |ln(GMR)|² ≤ k × σ̂_R², где k = 0.760
- Два отдельных RSABE-теста: натощак и после еды

### Метрики модели

| Метрика | Значение |
|---|---|
| Ожидаемый дизайн | REPLICATE_DESIGN |
| Ожидаемое решение RSABE | APPLICABLE |
| Ожидаемый N рандомизации | 56–72 |
| Ожидаемый эффект пищи | FASTING_AND_FED |

### Ожидаемый результат

```json
{
  "studyDesign": "REPLICATE_DESIGN",
  "rsabe": { "decision": "APPLICABLE", "lowerBound": 69.84, "upperBound": 143.19 },
  "foodEffect": "FASTING_AND_FED",
  "pkParameters": { "washoutDays": 10.0, "accumulationRisk": true }
}
```

## Сценарий 2: Иматиниб — стандартный перекрёстный дизайн

### Исходные данные

| Параметр | Значение |
|---|---|
| Препарат | Иматиниб (Gleevec®/Glivec®, генерик) |
| CV_intra | 20% |
| t½ | 18 ч |
| Эффект пищи | Нет (незначительный) |
| Показание | ХМЛ (хронический миелоидный лейкоз) |

### Методика

1. **Дизайн**: 2×2 перекрёстный (стандартный)
2. **Состояние**: Натощак (только)
3. **Washout**: 4 дня (5 × 18ч = 90ч)
4. **Выборка**: N ≈ 18–24 рандомизировано

### Цель

Доказать стандартную БЭ иматиниба. CV ≤ 30% → RSABE неприменимо → стандартные границы 80–125%.

### Как работает модель

- CV_intra = 20% ≤ 30% → стандартный crossover
- Границы: 80–125% (фиксированные)
- Owen's exact method для расчёта выборки

### Метрики модели

| Метрика | Значение |
|---|---|
| Ожидаемый дизайн | TWO_WAY_CROSSOVER |
| Ожидаемое RSABE | NOT_APPLICABLE |
| Границы | 80–125% |
| Ожидаемый N | 18–24 |

## Сценарий 3: Кабозантиниб — предельная вариабельность, параллельный дизайн

### Исходные данные

| Параметр | Значение |
|---|---|
| Препарат | Кабозантиниб (Cometriq®) |
| CV_intra | 55% |
| t½ | 55 ч |
| Особенности | Высокая межсубъектная вариабельность |

### Методика

1. **Дизайн**: Параллельный (две независимые группы)
2. **Обоснование**: CV > 50% — RSABE неприменимо (превышен порог 50%)
3. **Выборка**: N > 100 рандомизировано (увеличена из-за потери precision)

### Цель

При CV > 50% RSABE становится нецелесообразным. Параллельный дизайн исключает эффект переноса, но требует значительно большей выборки.

### Как работает модель

- CV = 55% > 50% → RSABE NOT_ALLOWED
- selectDesign() → PARALLEL_DESIGN
- Между-субъектная σ ≈ 1.5 × σ_within

### Метрики модели

| Метрика | Значение |
|---|---|
| Ожидаемый дизайн | PARALLEL_DESIGN |
| Ожидаемое RSABE | NOT_ALLOWED |
| N >> crossover | Да |

## Сценарий 4: Дазатиниб (t½=96ч) — критический washout

### Исходные данные

| Параметр | Значение |
|---|---|
| Препарат | Дазатиниб (Sprycel®) |
| CV_intra | 25% |
| t½ | 96 ч (4 дня) |
| Особенности | Очень длительный период полувыведения |

### Методика

1. **Дизайн**: 2×2 crossover
2. **Washout**: 20 дней (5 × 96ч = 480ч)
3. **Риск накопления**: ВЫСОКИЙ — AUC0-∞/AUC0-t должен быть ≤ 1.20

### Цель

Проверить правильность расчёта washout для препарата с очень длительным t½. Стандарт: 5 × t½ минимум. При t½ > 72ч — критическое предупреждение.

### Как работает модель

- t½ = 96ч > 72ч → критическое предупреждение "КРИТИЧНО"
- washoutHours = 5 × 96 = 480ч
- washoutDays = 480/24 = 20 дней

### Метрики модели

| Метрика | Значение |
|---|---|
| accumulationRisk | true |
| washoutDays | 20 |
| Уровень предупреждения | КРИТИЧНО |

## Сценарий 7: Мощность 90% — увеличенный размер выборки

### Исходные данные

| Параметр | Значение |
|---|---|
| Препарат | Гипотетический онкопрепарат |
| CV_intra | 25% |
| t½ | 12 ч |
| Статистическая мощность | 90% (вместо стандартных 80%) |

### Методика

1. Стандартный 2×2 crossover
2. Мощность увеличена до 90% по требованию регулятора
3. Сравнение выборки при 80% и 90%

### Цель

Показать, как требование повышенной мощности (90%) увеличивает необходимую выборку. Регуляторы иногда требуют 90% мощности для критических препаратов.

### Как работает модель

- Power = 0.90 → z_β увеличивается
- N_90% > N_80% (как правило на 30-40%)
- nScreened пересчитывается с учётом dropout

### Метрики модели

| Метрика | Значение |
|---|---|
| power | 0.90 |
| N_рандомизации (80%) | ~18 |
| N_рандомизации (90%) | ~24–26 |
| Прирост выборки | ~30–40% |

## Сценарий 8: Пограничный CV=30% — граница HVD

### Исходные данные

| Параметр | Значение |
|---|---|
| Препарат | Тест — пограничный препарат |
| CV_intra | 30.0% (ровно) |
| t½ | 12 ч |

### Методика

1. CV = 30% ≤ 30% → стандартный 2×2 crossover
2. RSABE НЕ применяется (≤30%)
3. Границы: стандартные 80–125%

### Цель

Проверка граничного условия: при CV ровно 30% модель должна НЕ применять RSABE. Это критически важное граничное условие по ЕЭК №85 и EMA.

### Как работает модель

```kotlin
cvIntra <= CV_THRESHOLD_HVD (30.0) → RSABEDecision.NOT_APPLICABLE
```

Строгое ≤ (не <): CV = 30.0% → NOT_APPLICABLE.

## Сценарий 9: Три компаратора — 3-way crossover

### Исходные данные

| Параметр | Значение |
|---|---|
| Препарат | Генерик A, Генерик Б vs Оригинал |
| CV_intra | 22% |
| t½ | 10 ч |
| Сравнений | 3 формуляции |

### Методика

1. **Дизайн**: 3-way crossover (3 периода, 3 последовательности)
2. Одновременное сравнение двух генериков с оригинальным препаратом
3. Выборка: эффективная σ уменьшается за счёт 3 периодов

### Цель

Продемонстрировать 3-way crossover для сравнения нескольких формуляций одновременно. Экономически выгоден при наличии двух генериков.

### Как работает модель

- THREE_WAY_CROSSOVER → σ_eff = σ × √(2/3)
- nPerSequence × 3 = nRandomized
- Washout между всеми тремя периодами

## Сценарий 10: Полный workflow README — CV=42%, t½=48ч, с едой

### Описание

Это эталонный тест-сценарий из README.md, демонстрирующий полный workflow расчёта для типичного орального онкопрепарата с высокой вариабельностью.

### Исходные данные

```json
{
  "drugName": "OralOncoDrug_REF",
  "cvIntra": 42.0,
  "halfLife": 48.0,
  "foodEffect": true,
  "power": 0.80,
  "alpha": 0.05,
  "dropoutRate": 0.15,
  "screenFailRate": 0.20
}
```

### Ожидаемый результат (по README)

| Параметр | Ожидание |
|---|---|
| studyDesign | REPLICATE_DESIGN |
| rsabe.decision | APPLICABLE |
| rsabe.lowerBound | ~69.84 |
| rsabe.upperBound | ~143.19 |
| pkParameters.washoutDays | 10.0 |
| pkParameters.accumulationRisk | true |
| foodEffect | FASTING_AND_FED |
| sampleSize.nRandomized | 56–72 |
| sampleSize.nScreened | 85–110 |

### Методика интерпретации

Этот сценарий полностью охватывает решение следующих задач:

1. **CV > 30%** → высоковариабельный препарат → реплицированный дизайн
2. **RSABE** применимо → расширенные масштабированные границы
3. **t½ = 48ч** → накопление! → washout = 10 дней
4. **Влияние пищи** → два отдельных исследования (натощак + после еды)
5. **N = 64–68** рандомизировано → N = 92–104 скринировано (dropout + screen-fail)

### Соответствие регуляторным требованиям

- **ЕЭК №85 (решение №85)**: RSABE для HVD разрешено, реплицированный дизайн обязателен
- **EMA CPMP/QWP/EWP/1401/98 Rev.1**: динамический RSABE-критерий
- **FDA Guidance 2003**: масштабированные границы для CV > 30%
